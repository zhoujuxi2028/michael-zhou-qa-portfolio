## 1. 概述

### 1.1 用例分类

| 类别       | 文件数 | 用例数 | 覆盖模块                               |
| ---------- | ------ | ------ | -------------------------------------- |
| API 层     | 4      | 27     | products, orders, auth, health/metrics |
| 横切关注点 | 3      | 13     | 认证保护路由, 错误处理, 并发访问       |
| 中间件     | 2      | 9      | 限流器, 安全头                         |
| 工具函数   | 3      | 12     | 基线对比, 泄漏检测, 趋势采集           |
| **合计**   | **12** | **61** | —                                      |

### 1.2 优先级定义

| 优先级 | 含义                   | 用例数 |
| ------ | ---------------------- | ------ |
| **P1** | 核心功能，阻塞发布     | 32     |
| **P2** | 重要功能，强烈建议通过 | 24     |
| **P3** | 辅助功能，发布前完成   | 5      |

### 1.3 公共前置条件

所有 Jest 集成测试共享以下前置条件：

| 条件          | 说明                                       |
| ------------- | ------------------------------------------ |
| Node.js ≥ 18  | `node -v` 确认                             |
| 依赖已安装    | `npm install` 完成                         |
| 无端口冲突    | 端口 3000 空闲（Supertest 不占用真实端口） |
| NODE_ENV=test | Jest 自动设置                              |
