# QA Scenarios in CI/CD Pipelines

在 CI/CD 流水线中，QA（质量保障）不再只是上线前的最后一道关卡，而是嵌入到每一个环节，通过自动化手段持续验证软件质量。以下是与 QA 相关的典型场景：

---

## 1. 代码提交与静态分析阶段

- **代码规范检查**：自动运行 Lint 工具（如 ESLint、Pylint、Checkstyle），检测编码风格与潜在语法错误。
- **静态应用安全测试（SAST）**：使用 SonarQube、Fortify 等扫描源码中的安全漏洞（如 SQL 注入、硬编码密钥）。
- **复杂度与重复度检测**：分析圈复杂度、代码重复率，阻止质量过差的代码合入主干。

## 2. 单元测试阶段

- **单元测试执行**：自动运行 JUnit、pytest、Mocha 等框架编写的单元测试。
- **覆盖率收集与门禁**：设定覆盖率阈值（如行覆盖 ≥ 80%，分支覆盖 ≥ 70%），不达标则流水线失败。
- **变异测试**（高级场景）：通过工具（PITest）注入突变，验证单元测试的有效性。

## 3. 集成与接口测试阶段

- **API 自动化测试**：使用 Postman/Newman、REST Assured、Karate 对微服务接口进行契约或集成测试。
- **契约测试**：基于 Pact 或 Spring Cloud Contract 验证服务间接口匹配，避免集成断裂。
- **数据库迁移与数据一致性测试**：在集成环境运行 DDL 变更，验证数据完整性及回滚脚本。

## 4. 组件与端到端测试阶段

- **UI 自动化测试**：借助 Selenium、Cypress、Playwright 模拟用户关键路径（登录、下单、支付）。
- **视觉回归测试**：使用 Percy、BackstopJS 比对 UI 截图，发现样式偏移。
- **组件级非功能测试**：对单个微服务进行短时压力或资源消耗基线测试。

## 5. 安全与合规专项阶段

- **软件成分分析（SCA）**：扫描依赖库中的已知 CVE 漏洞及许可证合规性（如 Black Duck、OWASP Dependency-Check）。
- **动态应用安全测试（DAST）**：对已部署的测试环境发起主动攻击模拟，发现运行时漏洞（如 OWASP ZAP、Burp Suite）。
- **容器镜像扫描**：检查 Docker 镜像中的系统漏洞、敏感文件（Trivy、Clair）。

## 6. 性能与稳定性测试阶段

- **烟雾性能测试**：在 CI 流程中快速运行短时压力（如 50 并发 × 30 秒），发现性能回归。
- **负载/压力测试**：在专门阶段使用 JMeter、k6、Gatling 模拟真实峰值流量，配合 APM（如 SkyWalking）定位瓶颈。
- **混沌工程实验**：在预发布环境注入故障（延迟、Pod 杀掉），验证系统弹性（Chaos Mesh、Gremlin）。

## 7. 部署与发布阶段（生产前）

- **冒烟测试套件**：部署到 staging 环境后立即运行核心用例集，确认基础功能正常。
- **蓝绿/金丝雀部署验证**：在新版本实例上运行健康检查与业务探针，自动比对新旧版本响应指标。
- **配置与环境一致性检查**：验证环境变量、证书、第三方服务连通性，防止配置漂移。

## 8. 生产环境持续质量保障

- **合成监控（Synthetic Monitoring）**：以固定频率（如每 5 分钟）调用关键 API 或 UI 流程，模拟用户行为，告警功能异常。
- **实时异常检测**：集成 Sentry、ELK 监控生产日志中的错误率，与 CI/CD 关联定位引入变更。
- **A/B 测试与功能开关**：逐步放量并对比业务指标（转化率、错误率），作为"质量反向门禁"。

---

## 核心价值

| 目标 | 说明 |
|------|------|
| **左移质量** | 缺陷发现越早，修复成本越低。CI 阶段失败即阻断，避免污染下游。 |
| **快速反馈** | 开发提交代码后几分钟内得到质量结果，缩短调试周期。 |
| **自动化回归** | 每次代码变更自动运行全部测试，消除手动回归的遗漏与延迟。 |
| **风险门禁** | 通过质量门禁（Quality Gate）强制满足测试覆盖率、漏洞等级、性能基线才允许发布。 |

实际工业流程中，企业往往根据自身成熟度选择上述场景的子集，并使用 Jenkins、GitLab CI、GitHub Actions、Azure Pipelines 等工具串联。
