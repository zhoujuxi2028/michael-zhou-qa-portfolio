# 登录页面需求文档

## 功能概述

登录页面提供用户身份验证入口，支持账号密码登录及密码找回流程。

---

## 页面元素

| 元素 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 用户名输入框 | `input[type=text]` | ✅ | 支持邮箱或用户名格式 |
| 密码输入框 | `input[type=password]` | ✅ | 内容以 `*` 遮蔽显示 |
| 登录按钮 | `button` | — | 提交表单，触发身份验证 |
| 忘记密码链接 | `a` / `button` | — | 跳转至密码找回流程 |

---

## 功能需求

### FR-LOGIN-001 用户名密码登录

| 属性 | 内容 |
|------|------|
| 描述 | 用户输入有效用户名和密码后，系统完成身份验证并跳转至首页 |
| 前置条件 | 用户已注册，账号处于激活状态 |
| 成功路径 | 输入正确凭据 → 点击登录 → 跳转首页，会话建立 |
| 失败路径 | 凭据错误 → 显示错误提示，停留登录页 |

**验证规则：**

| 规则 | 说明 |
|------|------|
| 用户名不可为空 | 提交时校验，显示 "用户名不能为空" |
| 密码不可为空 | 提交时校验，显示 "密码不能为空" |
| 凭据错误提示 | 显示 "用户名或密码错误"（不区分哪个字段错误） |
| 连续失败锁定 | 连续失败 5 次后，账号锁定 15 分钟 |
| 会话有效期 | 登录成功后 session 有效期 30 分钟（无操作自动过期） |

---

### FR-LOGIN-002 忘记密码

| 属性 | 内容 |
|------|------|
| 描述 | 用户点击"忘记密码"后，进入密码重置流程 |
| 触发方式 | 点击登录页"忘记密码"链接 |
| 流程 | 输入注册邮箱 → 系统发送重置邮件 → 用户点击链接 → 设置新密码 |

**验证规则：**

| 规则 | 说明 |
|------|------|
| 邮箱格式校验 | 不符合 email 格式时提示 "请输入有效的邮箱地址" |
| 邮箱未注册 | 显示 "如该邮箱已注册，您将收到重置邮件"（安全起见不区分） |
| 重置链接有效期 | 24 小时内有效，过期后提示重新申请 |
| 新密码规则 | 至少 8 位，含大小写字母和数字 |
| 重置后旧密码失效 | 密码重置成功后，所有已登录会话立即失效 |

---

## 安全需求

| 需求 | 说明 |
|------|------|
| 防暴力破解 | 连续失败 5 次锁定账号；支持 CAPTCHA 触发机制 |
| 防 SQL 注入 | 用户名/密码字段所有输入须参数化处理 |
| 防 XSS | 输入内容渲染前转义，禁止执行脚本 |
| HTTPS 传输 | 所有认证请求必须通过 HTTPS |
| 密码不明文存储 | 后端存储使用 bcrypt 哈希（salt rounds ≥ 12） |
| 登录日志 | 记录每次登录尝试（时间、IP、结果） |

---

## 非功能需求

| 指标 | 要求 |
|------|------|
| 响应时间 | 登录请求 < 2s（P95） |
| 可用性 | 页面可用率 ≥ 99.9% |
| 兼容性 | 支持 Chrome / Firefox / Safari / Edge 最新两个版本 |
| 移动端 | 响应式布局，支持 iOS / Android 主流机型 |

---

## 边界条件

| 场景 | 输入 | 期望结果 |
|------|------|----------|
| 用户名最大长度 | 256 字符 | 超出时截断或报错 |
| 密码最大长度 | 128 字符 | 超出时报错 |
| 密码最小长度 | 8 字符 | 不足时提示密码强度不足 |
| 空白字符用户名 | `"   "` | 视为空输入，提示不能为空 |
| 特殊字符用户名 | `admin'--` | 正常处理，不触发 SQL 注入 |

---

## 不在范围内（Out of Scope）

- 第三方 OAuth 登录（Google / GitHub）
- 多因素认证（MFA / 2FA）
- 单点登录（SSO）
