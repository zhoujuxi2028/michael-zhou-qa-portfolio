# 项目级需求台账模板

> **使用方法：** 复制本文件到 `<project>/docs/requirements/req-register.md`，将 `<PROJ>` 替换为项目简称（如 `PERF`、`K8S`、`SID`、`API`）。
> **ID 格式：** `<PROJ>-REQ-NNN`（三位流水号）
> **父文档：** Portfolio 级规则见 [`docs/requirements/README.md`](../../requirements/README.md)

---

## 1. 优先级速查

| 级别 | 含义 | Gate 影响 |
|------|------|-----------|
| P1 / Must Have | 核心功能，缺失则项目不可交付 | 阻断 Stage Gate |
| P2 / Should Have | 重要功能，影响完整度 | Stage 5 前必须闭环 |
| P3 / Could Have | 增值功能，时间允许时实现 | 不阻塞 |
| P4 / Won't Have | 本期明确不做 | 直接 Dropped |

---

## 2. 活跃需求台账（Active Requirements）

| REQ ID | GitHub Issue | 标题摘要 | 类型 | 优先级 | 提出日期 | 状态 | AC 编写 | 关联测试用例 |
|--------|--------------|----------|------|--------|----------|------|---------|-------------|
| `<PROJ>`-REQ-001 | #NNN | _(需求描述)_ | FR | P1 | YYYY-MM-DD | Proposed | ❌ | — |

**Acceptance Criteria 格式（在对应 Issue 中填写）：**

```
Given  [前置条件]
When   [触发动作]
Then   [期望结果]
And    [附加断言（可选）]
```
