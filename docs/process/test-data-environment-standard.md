# 测试数据与环境标准（复现一致性）

> 目标：降低“本地通过、CI 失败”。

## 1. 测试数据分层

| 层级 | 用途 | 变更策略 |
|------|------|----------|
| 基线数据（Baseline） | 冒烟/回归通用最小集 | 受控变更，需评审 |
| 场景数据（Scenario） | 业务流程、边界与异常 | 按用例演进 |
| 压测数据（Performance） | 大体量与高并发模拟 | 与性能脚本版本绑定 |

## 2. 环境一致性要求

| 维度 | 本地 | CI | Nightly/Soak |
|------|------|----|--------------|
| 依赖版本 | 必须可追踪 | 锁定版本 | 与 CI 同基线 |
| 启动顺序 | 按项目 CLAUDE 指南 | workflow 固化 | 与 CI 一致 |
| 配置源 | `.env.example` / 文档 | secrets + vars | 同 CI 策略 |
| 端口分配 | 遵循根 CLAUDE 表 | 冲突即 fail | 冲突即告警 |

## 3. 冲突预警与快速诊断

标准诊断入口：
- 端口占用：`lsof -i :<port>`
- 容器状态：`docker ps`
- 关键服务健康检查：项目 health endpoint / smoke 脚本

## 4. 执行规则

- 新增测试前先声明数据层级
- 新增依赖必须同步到依赖清单
- 测试失败必须标注原因分类：代码/环境/数据/依赖
