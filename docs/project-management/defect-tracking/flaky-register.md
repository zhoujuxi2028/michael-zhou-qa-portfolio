# Flaky Tests Register（稳定性治理登记表）

> 范围：Portfolio 跨项目 flaky 用例跟踪。  
> 规则：连续 N 次不稳定（默认 N=3）必须登记并指派 owner。

## 1. 分级标准

| 等级 | 判定 | Gate 影响 |
|------|------|-----------|
| P0 | 阻塞主线发布 / 核心路径 | BLOCKED |
| P1 | 高频波动，影响评审结论 | 条件阻塞 |
| P2 | 低频波动，可观测可绕行 | 不阻塞，限期治理 |

## 2. 活跃 Flaky 列表

| Flaky ID | 项目 | 用例/Job | 分级 | 不稳定窗口 | 最近状态 | 原因分类 | Owner | 截止日期 | 状态 |
|----------|------|----------|------|------------|----------|----------|-------|----------|------|
| _(暂无)_ | | | | | | | | | |

原因分类统一使用：`代码缺陷 / 环境波动 / 数据问题 / 依赖问题`。

## 3. 修复状态区分

| 状态 | 定义 |
|------|------|
| Retry-Pass | 仅通过重试恢复，未修复根因 |
| Fix-Verified | 根因修复并稳定通过 |
