# 依赖风险修复 SLA（供应链安全）

> 目标：高风险依赖“可量化、可追踪、有限时”修复。

## 1. 风险等级与时限

| 风险等级 | 示例 | 修复 SLA |
|----------|------|----------|
| Critical / P0 | 远程执行、凭据泄漏、广泛可利用 | 24 小时内缓解，72 小时内修复 |
| High / P1 | 高危漏洞，存在可行利用链 | 7 天内修复 |
| Medium / P2 | 中危漏洞 | 30 天内修复或登记 Waiver |
| Low / P3 | 低危漏洞 | 90 天内修复或接受风险 |

## 2. 执行流程

1. 扫描发现 → 生成 Issue + 关联 Defect ID  
2. 判定等级 → 设置 SLA 截止时间  
3. 修复验证 → CI 安全扫描通过  
4. 关闭回写 → 更新 defect register 与变更日志

## 3. Waiver 规则

- P0 不允许 Waiver
- P1 仅限非核心路径且需审批
- P2/P3 可按 `waiver-policy.md` 执行

## 4. 追踪字段（最小集）

| 字段 | 说明 |
|------|------|
| 组件名 | 受影响依赖 |
| 版本 | 当前版本与目标版本 |
| 风险级别 | CVSS / 内部等级 |
| 截止时间 | SLA 到期日 |
| 关联 Issue/PR | 闭环追踪 |
